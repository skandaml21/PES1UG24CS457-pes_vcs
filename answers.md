## Phase 5: Branching

Branching allows multiple lines of development in a version control system.  
Each branch points to a specific commit, enabling independent work without affecting the main codebase.  
In Git, branches are lightweight pointers to commits, making switching and merging efficient.

---

## Phase 6: Checkout

Checkout is used to switch between commits or branches.  
It updates the working directory and index to match the selected commit.  
This allows users to navigate different versions of the project.

---

## Phase 6: Garbage Collection

Garbage collection removes unused or unreachable objects from the repository.  
Over time, objects may become orphaned due to operations like rebasing or deleting branches.  
This process helps in saving storage and maintaining repository efficiency.
